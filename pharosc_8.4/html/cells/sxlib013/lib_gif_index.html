Viewable GIF files for the Alliance sxlib. This release contains the cells that can be characterised by the scripts in the library: 61 cells in all, combinatorial functions with 1-4 inputs. Each cell has a typical timing arc and input pin capacitance in 0.13µm generic technology, along with its leakage and dynamic power, size, layout, and the transistor schematic. Transistor sizes are given in lambda, where for this 0.13µm technology, lambda=0.055µm. The cell height is 100 lambda (5.5µm) with power supply rails 12 lambda (0.66µm) wide. The maximum P and N transistor widths are 40λ and 34λ respectively.

Note that this lambda value is half the value used in the Alliance sxlib layout. This means that the transistor widths in the cells' schematics are twice the value that will be measured in Graal … e.g. a 40λ transistor will be 20&lambda wide in Graal. Both are 2.2µm wide after conversion to CIF.

The sxlib has been drawn with its own layout rules which means that the 0.13µm layout has plenty of DRC errors. The ssxlib has been derived from the sxlib and has DRC clean layout.
